# Session lesson — adaptive replies + 7-day conversation alignment audit (2026-07-01)

## Trigger

Moufadal explicitly complained that Hermes was formatting almost every Telegram answer the same way (`📌 Verdict`, `🧭 Où on en est`, `✅ Actions faites`, etc.), even when the prompt was only a short question, a small search, or a live check-in.

He then asked for a 7-day audit that includes session/compaction friction, slash/background issues, unclear voice-style prompts, Obsidian usefulness, a corpus Markdown for another AI, and a mega-prompt.

## Durable lesson

This is not only a memory preference. It is a skill-library lesson: conversation-review work must convert style/friction corrections into reusable operational rules.

Recommended response tiers for Telegram:

- **T0 Flash** — presence checks, yes/no, tiny status: 1–4 lines, no Projet Clair sections.
- **T1 Compact** — small action or guided command: direct verdict + next step, short bullets.
- **T2 Standard** — bounded diagnostic: facts, interpretation, limitation, next action.
- **T3 Projet Clair** — serious project/dev/VPS/scraping/long research/risky system work: full structured report with proof paths and rollback.

## Obsidian finding

Obsidian distillations are valuable for speed and semantics: they surface stable preferences, repeated corrections, decisions, and project context faster than raw transcripts. But draft/autosave notes are not ground truth. For durable skill/memory changes, cross-check with `session_search`, artifacts, or current user confirmation.

## 7-day audit artifact pattern

For future “analyse nos conversations / on doit être en phase” tasks, produce these durable artifacts under a dated directory:

- `conversations_7jours_pour_ia.md` — redacted user/assistant corpus suitable for another AI.
- `obsidian_notes_7jours.md` — relevant distilled notes.
- `analyse_alignement_7jours.md` — Hermes’ own diagnosis.
- `MEGA_PROMPT_ANALYSE_EXTERNE.md` — prompt for another AI.
- `index.html` — clickable report/index.
- `TASK_LEDGER.md` + logs when the run is backgrounded.

## Pitfall prevented

Do not treat “user dislikes response format” as only a memory update. Patch the delivery/review skills so future sessions already know how to choose the right output shape.
